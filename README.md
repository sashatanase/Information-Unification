## How it works
- **One issue per change.** Open a new issue with the built-in template; it prompts for
  the problem, fix options, impact, done-when, type, and priority. See
  [TASK_TEMPLATE.md](TASK_TEMPLATE.md) for what each section means.
- **Labels** keep the board filterable:
  - **Asset** — `website`, `linkedin`, `github`, `resume-uxr`, `resume-pm`
  - **Type** — `Content`, `Structure`, `Visual`, `Research`, `Ops`, `Technical`
  - **Priority** — `priority: high` / `medium` / `low`
- **@claude** can act on an issue — mention `@claude` in the issue or a comment and it
  will do the work and open a PR where applicable.
